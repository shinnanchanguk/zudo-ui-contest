# Dev Loop Checklist
> Created: 2026-04-23T12:07+09:00
> Status: IN_PROGRESS

## 요구사항 요약
하나의 링크에서 모든 학생 제출작을 모아 보고, 카드 클릭으로 각 디자인을 바로 체험할 수 있는 갤러리 페이지를 만든다. 선생님은 이 한 URL만 전교생에게 공유하면 됨.

## 체크리스트

### 구현 검증
- [ ] `/gallery` 라우트가 production(`https://zudo-ui-contest.vercel.app/gallery`)에서 200 반환
- [ ] 페이지 제목/설명이 한국어로 명확히 표시됨 ("제출작 갤러리" 등)
- [ ] GitHub open PR 데이터를 실제로 가져와서 렌더 — fetch 실패 시 fallback UI 노출 (페이지 깨지지 않음)
- [ ] 제출작이 0개일 때 빈 상태 메시지 표시 ("아직 제출작이 없어요" 류)
- [ ] 제출작이 1개 이상일 때 카드 그리드로 렌더 (모바일 1열 / 태블릿 이상 다열)

### 카드 콘텐츠 검증 (테스트용 PR 1개 생성 후)
- [ ] 카드에 PR 제목 표시
- [ ] 카드에 작성자 GitHub username + 아바타 표시
- [ ] 카드에 👍 reaction 수 표시 (0이어도 0으로 표시)
- [ ] **[체험하기]** 버튼 클릭 시 해당 PR의 Vercel preview URL로 이동 (새 탭)
- [ ] **[PR 보기]** 링크 클릭 시 GitHub PR 페이지로 이동 (새 탭, 투표/댓글용)
- [ ] preview URL이 실제로 200 반환하고 mock 앱 화면 표시

### 정렬 및 동작 검증 (테스트 PR 2개로)
- [ ] 👍 많은 PR이 위에, 같은 수면 최신이 위에 (테스트 PR 2개에 reactions 다르게 주고 확인)
- [ ] CLOSED PR은 갤러리에서 제외됨 (테스트 PR 1개를 닫고 확인)
- [ ] Vercel bot이 아직 코멘트 안 단 PR은 "준비중" 상태 표시 또는 우아하게 처리 (preview URL 없을 때)

### UX 검증
- [ ] 모바일 360px 너비에서 카드가 가로 스크롤 없이 표시됨
- [ ] /m 홈 화면에서 갤러리로 가는 링크 **없음** (학생 fork 화면이 갤러리를 노출하면 어색하므로 학생 UI는 그대로 두기)
- [ ] README에 갤러리 URL 추가됨

### 안전 확인
- [ ] GitHub API 호출이 unauthenticated (토큰 없이) — 운영 secret 노출 없음
- [ ] fetch revalidate=60으로 설정 → 1분 캐시 → GitHub rate limit 안전
- [ ] 테스트 PR/브랜치 정리 (close + delete)
